## 三、比赛目标与赛题方向

- **核心目标**：在**消费级 GPU**上实现**百万 Token 推理**，且 **KV Cache < 6GB**。
- **技术方向**（可多选或组合）：
  - 稀疏算子融合与加速
  - 跨层编译优化
  - 混合注意力架构（SALA）推理加速
- **基座模型**：必须基于官方提供的 **MiniCPM-SALA** 模型进行优化，**不得提交或替换任何基座模型**（可做量化、剪枝等预处理，但基座来源唯一）。

## 五、评测环境与资源限制

（以官方 Toolkit 说明为准）

- **提交包大小**：全部提交文件总大小 **不得超过 2GB**。
- **单次评测资源**：
  - CPU：最多 **20 核**
  - 内存：**128 GiB**
  - 单次任务最长运行时间：**5 小时**（不含排队时间）
- **评测硬件**：统一使用 **NVIDIA 高端 RTX PRO 单卡 GPU**。

---

## 六、评测维度与规则

### 6.1 正确性评测

- **目的**：确保优化后的推理代码不损害模型能力。
- **数据**：
  - 公开集：`perf_public_set.jsonl`（选择题、信息提取等，多长度）。
  - 基准分：**80 分**（原始模型约 82±2 分）。
- **判定**：评测会计算相对该基准分的**相对分数**（如 `overall_accuracy`），用于判断是否“掉点”。
- **防作弊**：另有私有集 `perf_private_set.jsonl`，与公开集长度分布与任务类型一致，用于检查公开/私有集表现是否异常，保证公平。

### 6.2 速度评测

- **方式**：使用官方 `bench_serving.sh`（基于 sglang 的 bench_serving），在 **3 档并发**下分别跑完全部请求，记录 **Benchmark Duration**。
- **三档**：
  - **S1**：并发 = 1
  - **S8**：并发 = 8
  - **Smax**：不设并发上限
- **输入格式**：固定输入与输出，通过 `question` 与 `model_response` 等字段构造 JSONL；比赛用速度数据集不公开，题目长度分布以赛题说明为准。

## 七、提交规则

### 7.1 提交物形式

- 将所有代码及资源打成 **`.tar.gz`**，**必须包含**：
  - **`prepare_env.sh`**（必选）：环境构建脚本。
  - **`prepare_model.sh`**（可选）：模型预处理脚本。
  - 其他代码/资源按需组织。

### 7.2 执行流程

1. **环境构建**：平台先起基础环境，再**自动执行**选手的 `prepare_env.sh`，安装依赖与配置。
2. **模型预处理**（若提供）：随后平台调用 `prepare_model.sh`，对**平台提供的原始模型**做预处理，并输出到**平台指定路径**，供后续推理使用。

`prepare_model.sh` 必须支持：

```bash
bash prepare_model.sh --input <原始模型路径> --output <处理后模型路径>
```

- **包管理**：基础环境使用 **`uv`**，安装 Python 包时需使用 **`uv pip install`**，并在本地充分测试后再提交。
- **旧方式**：此前通过 wheel 提交的选手，需将逻辑迁移到 `prepare_env.sh` + `prepare_model.sh`，**旧的 wheel 提交方式不再支持**。

## 八、禁止与约束

- **禁止替换基座模型**：只能基于官方 MiniCPM-SALA 优化，不得提交或替换基座模型本身。
- **提交包 ≤ 2GB**：所有提交文件总大小不得超过 2GB。
- **正确性门槛**：优化后模型在正确性评测上的相对表现需满足赛事要求（以官方说明为准），否则可能影响成绩或资格。
- **公平性**：禁止针对公开正确性集刷分；私有集用于交叉验证，异常差距将影响成绩。

## 十、规则小结（速查）

- **做什么**：在指定硬件上对 **MiniCPM-SALA** 做推理优化，追求百万 Token、KV Cache < 6GB，兼顾正确性与速度。
- **交什么**：`.tar.gz`，内含必选 `prepare_env.sh`、可选 `prepare_model.sh` 及自备代码/资源，总大小 ≤ 2GB。
- **怎么评**：正确性（相对 80 分基准 + 私有集校验）+ 速度（S1/S8/Smax 三档 Benchmark Duration）。
- **注意**：不换基座、用 `uv pip`、按新提交方式迁移，并关注官网与 Toolkit 页面的最新补充规则。

以上内容均来自官网与 SOAR-Toolkit 公开文档；若与官网最新说明不一致，**以官网及平台当前规则为准**。
